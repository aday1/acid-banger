OSC reference (fork)

This page lists IP/port roles, which OSC addresses the app sends and receives, and how the fields in the OSC panel map to them. You must run bridge/server.mjs (see bridge README) and enable Connect OSC bridge in the app.

Data flow (defaults)
TouchOSC / Max / etc. → UDP to your PC port 57121 → Node bridge → WebSocket ws://127.0.0.1:8765 → this browser tab.
Outgoing from the app: tab → WebSocket → bridge → UDP to the target IP:port you set (e.g. 127.0.0.1:9000).

Where to set IP and ports (OSC panel)

OSC panel label Typical value What it is
Bridge machine (host) 127.0.0.1 Host for the WebSocket connection to Node. Use the PC’s LAN IP if the bridge runs on another machine. You can put a full URL here, e.g. ws://127.0.0.1:8765, instead of host + port.
WebSocket port 8765 Must match the bridge env OSC_WS_PORT (default 8765). This is not the port TouchOSC uses.
Bridge OSC UDP listen port 57121 Reminder label only: external OSC apps must send UDP to this port on the machine running the bridge. Match OSC_UDP_PORT (default 57121).
Target IP / host 127.0.0.1 UDP destination for messages from the app (/acid/step, /acid/bpm). Often localhost or another app on your LAN.
Target UDP port 9000 (example) UDP port on that host. Your DAW or OSC tool must listen here for outgoing acid-banger traffic.

Bridge environment variables (optional)

When starting server.mjs:

OSC_UDP_PORT=57121   # UDP listen (OSC into the bridge)
OSC_WS_PORT=8765       # WebSocket port for the browser

Incoming OSC (into the browser)

Any OSC message received on the bridge’s UDP listen port is forwarded to the page as one WebSocket text frame of JSON: 

{"address":"/your/address","args":[1,2,3]}

The OSC in (last message) line in the app shows a short preview of every message. The sequencer only acts on tempo from addresses that match the rules below.

OSC address Arguments Effect in the app
/acid/bpm One number (float): BPM Sets the BPM dial, clamped to the app’s tempo range (70–200).
Any address ending with /bpm
(e.g. /live/set/tempo/bpm)		 First argument: BPM as a number Same as above: first float is applied as BPM (clamped).
Any other address Any No built-in action; preview only in OSC in (last message).

Configure your sender: destination IP = machine running the bridge (e.g. 127.0.0.1 on the same PC, or your PC’s Wi‑Fi IP from a phone). Destination UDP port = OSC_UDP_PORT (default 57121).

Outgoing OSC (from the browser)

When the WebSocket is connected, the page can ask the bridge to send UDP packets. The built-in toggles only send these addresses to your Target IP and Target UDP port:

OSC address When it is sent Arguments
/acid/step While Emit OSC /acid/step is on, each 16th step One float: current step index 0–15 (transport division in the app).
/acid/bpm While Emit OSC /acid/bpm is on, when BPM changes One float: current BPM.

The Node bridge encodes all arguments as OSC floats. There is no UI yet to define custom addresses; adding more would require a code change in the app (or another layer listening on the same UDP port).

WebSocket command (for custom tools)

From any WebSocket client connected to the bridge, you can send UDP by JSON:

{"cmd":"oscSend","address":"/foo","args":[1,2],"remoteHost":"127.0.0.1","remotePort":9000}

The bridge forwards one OSC message with float arguments to remoteHost:remotePort.

Quick checklist

  • Bridge running: console shows UDP and WebSocket ports.
  • App: Connect OSC bridge on; WebSocket host/port match the bridge.
  • Into the app: send OSC UDP to bridge machine : OSC_UDP_PORT (57121).
  • Out of the app: set target host/port; enable emit toggles for /acid/step and/or /acid/bpm.
  • In-browser visuals: use the built-in 303+909 visual panel and camera/record controls.
